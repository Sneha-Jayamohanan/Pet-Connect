📡 Exam Phone Detection System (IoT)

The Exam Phone Detector is a smart IoT-based surveillance system designed to identify the presence of unauthorized mobile devices in restricted environments such as examination halls. The project addresses a real-world problem—academic malpractice—by leveraging wireless signal detection, embedded systems, and intelligent alert mechanisms.

🔍 Project Overview

In modern examination settings, preventing the use of mobile phones is a major challenge. Traditional methods like manual checking are inefficient, time-consuming, and prone to human error. This project introduces an automated, scalable, and real-time solution using IoT technology to detect active mobile devices based on radio frequency (RF) emissions.

The system continuously monitors the surrounding environment for RF signals typically emitted by mobile phones during communication activities such as calls, messages, or data transmission. Once a signal is detected, the system immediately triggers alerts and logs the event for further action.

⚙️ Core Components & Technologies

  • Microcontroller (ESP32 / Arduino): Acts as the brain of the system, processing sensor inputs and controlling outputs.
  • RF Detector Module: Detects electromagnetic signals emitted by mobile phones.
  • PIR Sensor (Optional Layer): Adds human presence detection for smarter filtering and reduced false alerts.
  • Buzzer & LED Indicators: Provide instant local alerts when a device is detected.
  • IoT Connectivity (Wi-Fi): Sends real-time notifications to a monitoring system or mobile device.
  • Cloud / App Integration: Enables remote monitoring, logging, and analytics.

🧠 Working Principle

  • The RF detector continuously scans the environment for mobile signal frequencies.
  • When a signal is detected, it sends a HIGH signal to the microcontroller.
  • The microcontroller validates the signal and triggers:
    • Visual alert (LED)
    • Audible alert (buzzer)
    • IoT notification (via Wi-Fi)
  • The system can also log timestamps and detection frequency for analysis.
  • Optional layers (like PIR sensors) help ensure detection is relevant to human presence, improving accuracy.

🚀 Key Features

  • Real-Time Detection & Alerts
  • Wireless Monitoring via IoT
  • Scalable for Large Exam Halls
  • Low Cost and Easy Deployment
  • Layered Security Approach (RF + Presence Detection)
  • Data Logging for Post-Exam Analysis

💡 Innovation & Impact

This project goes beyond a simple detector—it creates a smart surveillance ecosystem. By combining hardware sensing with IoT-based communication, the system provides both immediate alerts and long-term monitoring capabilities.

It ensures:

  • Fair examination environments
  • Reduced reliance on manual invigilation
  • Increased transparency and accountability

🔮 Future Enhancements

  • Machine learning-based signal classification to reduce false positives
  • Integration with CCTV systems for visual verification
  • Mobile app dashboard for live monitoring
  • Geo-fencing and zone-based detection
  • Cloud analytics for pattern recognition

🏁 Conclusion

The Exam Phone Detector demonstrates how IoT can be applied to solve practical institutional problems efficiently. It blends hardware, networking, and intelligent processing into a cohesive system that is both impactful and scalable. This project reflects a strong foundation in embedded systems, IoT architecture, and real-world problem-solving.

🧠 ECG-Based Heart Attack Prediction Using Deep Learning

This project focuses on building an intelligent and reliable system for early prediction of heart attacks using Electrocardiogram (ECG) signals. Cardiovascular diseases remain one of the leading causes of death worldwide, and timely detection plays a critical role in saving lives. The primary objective of this project is to leverage deep learning techniques to analyze ECG data and identify patterns that indicate a high risk of cardiac events.

📊 Project Overview

The system is designed to process raw ECG signals and automatically extract meaningful features using neural networks, eliminating the need for manual feature engineering. By learning from large-scale physiological datasets, the model can distinguish between normal and abnormal heart activity and predict potential heart attack risks with high accuracy.

💡 Motivation

Traditional methods of heart disease detection often rely on manual interpretation by medical professionals, which can be time-consuming and prone to human error. This project aims to assist healthcare systems by providing a fast, scalable, and automated solution that enhances diagnostic accuracy and enables early intervention.

⚙️ Approach

The project follows a structured deep learning pipeline:

  • Data Collection: ECG datasets (such as MIT-BIH or similar clinical datasets) are used.
  • Preprocessing: Noise removal, normalization, and segmentation of ECG signals to ensure clean input.
  • Feature Learning: Deep learning models (such as CNNs, LSTMs, or hybrid architectures) automatically learn temporal and spatial patterns in ECG signals.
  • Model Training: The model is trained on labeled data to classify or predict heart attack risk.
  • Evaluation: Performance is measured using metrics like accuracy, precision, recall, and F1-score.
  • Prediction System: A final interface or module that takes ECG input and outputs risk predictions.

🚀 Key Features

  • Automated ECG signal analysis using deep learning
  • Early detection of abnormal cardiac patterns
  • High accuracy through data-driven learning
  • Scalable and adaptable to real-world healthcare systems
  • Potential integration with IoT or wearable devices

💻 Technologies Used

  • Python for development
  • Deep Learning frameworks (TensorFlow / PyTorch)
  • Signal processing techniques for ECG preprocessing
  • Data visualization and analysis tools

🏥 Applications

  • Early diagnosis of heart-related conditions
  • Clinical decision support systems
  • Integration with wearable health monitoring devices
  • Remote patient monitoring in telemedicine

🔮 Future Enhancements

  • Integration of explainable AI techniques (e.g., Grad-CAM) to improve trust and interpretability
  • Real-time ECG monitoring using IoT devices
  • Expansion to predict multiple cardiovascular conditions
  • Deployment as a web or mobile healthcare application

🏁 Conclusion

This project demonstrates how artificial intelligence can significantly improve healthcare outcomes by enabling early detection and continuous monitoring of heart conditions. By combining deep learning with biomedical signal processing, the system has the potential to assist doctors, reduce diagnostic delays, and ultimately save lives.

🐾 PetConnect – Project Overview

🔹 Implementation Summary

The project is developed using core web technologies:

  • HTML for structuring the content and layout of the website.
  • CSS for styling, responsiveness, and modern UI design.
  • Basic Web Design Principles to ensure accessibility and ease of navigation.

Key Features Implemented:

  • Structured homepage introducing the platform
  • Sections for pet listings (dogs, cats, etc.)
  • Clean navigation system for smooth user experience
  • Call-to-action elements like "Adopt Now" or "View Pets"
